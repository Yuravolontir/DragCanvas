
## 🎯 КАК ПОПРОБОВАТЬ:


## 🔄 ПОТОК ДАННЫХ (пошагово):

### Шаг 1: Юзер НЕ залогинен
```
StatusBar: ❌ Не залогинен (красный)
AuthContext.user = null
localStorage.user = (нет)

ProtectedRoute:
  const { user } = useAuth();  // user = null
  if (!user) → <Navigate to="/login" />

Результат: РЕДИРЕКТ НА /LOGIN
```

### Шаг 2: Юзер вводит пароль
```
Login component:
  1. Проверяет: email === 'test@test.com' && password === '123456'
  2. Если верно → вызывает login(mockUser)

AuthContext.login():
  1. setUser(mockUser)           // user = { id: 1, email: '...' }
  2. localStorage.setItem(...)   // сохраняет в браузере

Результат: ЮЗЕР ЗАЛОГИНЕН
```

### Шаг 3: Юзер ЗАЛОГИНЕН
```
StatusBar: ✅ Залогинен как: test@test.com (зелёный)
AuthContext.user = { id: 1, email: 'test@test.com' }
localStorage.user = "{ id: 1, email: 'test@test.com' }"

ProtectedRoute:
  const { user } = useAuth();  // user = { id: 1, email: '...' }
  if (!user) → НЕ СРАБАТЫВАЕТ!

Результат: ПОКАЗЫВАЕТ СТРАНИЦУ
```

## 🎓 ГЛАВНОЕ ПОНЯТЬ:

| Компонент | Что делает | Откуда берёт данные |
|-----------|------------|---------------------|
| **MySQL** | Хранит всех юзеров | - (это база данных) |
| **Backend** | Проверяет пароль в MySQL | Из MySQL |
| **AuthContext** | Хранит текущего юзера | Из localStorage при загрузке |
| **ProtectedRoute** | Проверяет доступ | Из AuthContext |
| **Login** | Логинит юзера | Вызывает AuthContext.login() |

---

## ⚠️ ВАЖНО:

- **AuthContext НЕ проверяет базу данных!**
- **AuthContext только ХРАНИТ кто залогинен**
- **Backend проверяет базу данных** (пока нет, будет потом)

### Сейчас (без backend):
```
Login → проверяет в коде (if email === 'test@test.com')
```

### Потом (с backend):
```
Login → отправляет на Backend
Backend → проверяет MySQL
Backend → возвращает токен
Login → сохраняет в AuthContext
```
